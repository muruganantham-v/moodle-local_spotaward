Spot Award Plugin

This document describes the local_spotaward plugin, including its business flow, user roles, features, settings page, notifications, reporting, and certificate process.

Prepared from the current codebase in this workspace.

Plugin Component local_spotaward
Release 1.1.0
Plugin Version 2026042212
Maturity Beta

1. Purpose

The Spot Award plugin manages recognition nominations for students inside Moodle. It allows mentors or nominators to create nominations, routes them to a Program Manager for review, passes approved cases to the SS Team for processing, supports certificate generation and distribution, and provides dashboards and performance reporting.

2. High-Level Features

Nomination Management

  • Course-based nomination form for students
  • Draft entry workflow before final submission
  • Multiple students per submission
  • Automatic award description generation by category and module

Review Workflow

  • Program Manager approval or rejection per student
  • Approve-all shortcut for pending items
  • Status tracking and history
  • Role-based dashboard visibility

SS Team Processing

  • Continue processing approved nominations
  • Close rejected tickets
  • Share PR documents to admin
  • Close complete records

Certificates and Reporting

  • Beautiful Certificate template integration
  • Generate, view, download, and bulk-download certificates
  • Email certificates to students
  • Course and student performance reports

4. Main Business Flow

  1. Nominator opens the Nomination Form
    The form is available to users who can access the plugin and have the nominator role.
  2. Nominator selects course and students
    Students are loaded dynamically based on the selected course, and program managers are also loaded for that course.
  3. Nominator adds draft entries
    Each draft entry captures the student set, module, award category, professional, award description, and selected Program Manager.
  4. Shared draft values stay fixed
    After the first draft entry is added, the form keeps the same course, module, professional, and Program Manager for later draft additions until Clear Draft is used.
  5. Nominator submits nominations
    All draft entries are persisted into nomination tables, and a Program Manager notification is sent.
  6. Program Manager reviews each student item
    The Program Manager can approve individual items, reject individual items with reason, or approve all pending items.
  7. Approved items move to SS Team
    The SS Team receives notification and can continue the process.
  8. Certificates are generated for approved items
    Once items move into SS Team progress, certificate generation is available through the configured Beautiful Certificate template.
  9. SS Team shares PR documents to Admin
    The SS Team can upload a PR document and optionally attach certificates when notifying the configured admin recipient.
  10. SS Team closes tickets or closes the record
    Rejected tickets can be closed individually, and a full nomination record can be closed when processing is complete.
  11. Students can receive certificates by email
    The plugin can email certificates to all approved students from the completed nomination.

6. Nomination Form Details

The nomination form contains these main fields:

  • Course
  • Students
  • Module
  • Award Category
  • Professional
  • Program Manager
Important current behavior: once a draft exists, the form locks Course, Module, Professional, and Program Manager until the user clicks Clear Draft.

Additional form behavior

  • Students are loaded dynamically through AJAX after course selection.
  • Program Manager options are loaded automatically for the selected course.
  • The first Program Manager in the returned list is pre-selected.
  • For course names containing 24024C, the professional is forced to Embedded Professional.
  • Award descriptions are auto-generated from category and module when a template exists.
  • If no generated description exists, raw description content can still be preserved for certificate output.

7. Submission Review and Status Flow

The plugin uses a nomination-level status and an item-level status per student.

Status Meaning
pending New submission waiting for Program Manager review
underreview Review is in progress
ssteamprogress Approved and currently being handled by the SS Team
rejected Rejected by Program Manager, with reason captured
closed Item or overall record closed after processing

Program Manager actions

  • Approve one student item
  • Reject one student item with rejection reason
  • Approve all pending items at once
  • View item-level learner report from the submission screen

SS Team actions

  • View all certificates for approved students
  • Download all certificates
  • Email certificates to students
  • Share PR document to configured admin
  • Close rejected tickets
  • Close the complete record

8. Performance Reports

The report module supports both course-level and learner-level reporting. It can be opened from the top-level report page or from the submission detail screen for a specific student item.

Report highlights

  • Course selector for authorized users
  • Activity type filter
  • Student-level detailed report
  • Course-level summary report
  • Activity categories such as assignments, projects, quiz, attendance, class works, and template programs
  • Completion and percentage-based reporting logic

9. Certificate Features

Certificate generation is integrated with the Beautiful Certificate plugin. The Spot Award plugin selects a configured Beautiful Certificate template, fills placeholders with nomination and student data, and stores generated PDFs in Moodle file storage.

Certificate capabilities

  • Generate one certificate per approved nomination item
  • View single certificate inline in browser
  • Download single certificate
  • View all certificates in merged PDF form
  • Download all certificates as ZIP
  • Email certificates to students
  • Attach certificates to admin PR communication if selected

Certificate placeholders

The settings page exposes certificate placeholder mappings such as:

  • Student name, first name, last name, email, username
  • Course name and shortname
  • Module name
  • Professional
  • Total students count
  • Award category and award description
  • Nomination date and issue date
  • Nominated by and presented by
  • Nominator and Program Manager email

14. Operational Notes

Menu visibility: the plugin can place a Spot Award link in Moodle top navigation when enabled.
Course restrictions: nominators can only nominate within courses they are allowed to use, and students are loaded only from those courses.
Program Manager selection: if a specific Program Manager role is configured, those users are used; otherwise the plugin falls back to teachers and editing teachers in the course.
Certificate dependency: certificate generation depends on the Beautiful Certificate plugin and a configured template.
